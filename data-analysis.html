dataanalysis

collecting > processing > reporting

Designing and implementing test methods in production is not the only aspect of good quality control. It's also very important to collect and analyze all quality data which emerges to continusly improve the processes. This can start by acivities as simple as talking to the relevant operators and ask for theier feedback. They are often the best souce of information. But there are also very ellaborate methods of collecting, analyzing and reporting automation.

Our products final test has roughly about 200-300 test points. analyzing if the test method is applicable for every test point would be a tidious task. Luckily statistical software like minitab allows for heavy automation through scripting. If thats not always possible one could use windows webdriver to automate almost every windows software. See my projects here.

Analyzing and plotting data is often done in python through the usual librarys like pandas and matplotlib. I use them as well to get quick and easy infomration and help for reporting. But not all your colluegues are sopthistcated programmers. So I also like to provide use friendly programms with a UI which the collugues can use.
